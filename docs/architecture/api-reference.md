# Q-Rate Lite: API Reference

All endpoints are served under the `/api` prefix. The base URL in production is `https://qrate-lite.vercel.app/api`.

## Authentication

The API uses two authentication mechanisms:

| Type | Used By | Header |
|---|---|---|
| **Supabase JWT** | Café owners, staff, superadmin | `Authorization: Bearer <supabase_jwt>` |
| **Customer JWT** | Customer feedback sessions | Passed in request body (not a header) |

Customer JWTs are short-lived (15 min), scoped to a specific café (`aud: cafe-{id}`, `iss: qrate-customer`), and generated by the `/auth/request-feedback-link` endpoint.

### `GET /api/`

**Response `200 OK`:**
```json
{ "status": "ok" }
```

## Auth (`/api/auth`)

### `POST /api/auth/request-feedback-link`

Generates a customer JWT for a given phone number and café, then sends the feedback link via WhatsApp.

**Rate limit:** 10 requests / minute (per IP).

**Query Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `phone` | string | Yes | Customer's 10-digit Indian phone number. |
| `cafe_id` | integer | Yes | The ID of the café. |

**Response `200 OK`:**
```json
{
  "token": "<jwt>",
  "redirect_url": "https://qrate-lite.vercel.app/feedback?token=<jwt>",
  "whatsapp_response": { ... }
}
```

**Errors:**
- `429 Too Many Requests` — Rate limit exceeded.

---

### `GET /api/auth/verify`

Verifies and decodes a customer JWT.

**Query Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `token` | string | Yes | The customer JWT to verify. |

**Response `200 OK`:**
```json
{
  "phone": "9876543210",
  "cafe_id": 42
}
```

**Errors:**
- `401 Unauthorized` — Token is invalid or expired.

## Feedback (`/api/feedback`)

### `POST /api/feedback/submit`

Submits a customer's feedback. Issues a coupon automatically. Idempotent — submitting again with the same token returns the existing coupon.

**Rate limit:** 5 requests / minute (per IP).

**Request Body:**
```json
{
  "token": "<customer_jwt>",
  "rating": 4,
  "comment": "Great coffee!",
  "marketing_opt_in": true
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `token` | string | Yes | Valid customer JWT from `/auth/request-feedback-link`. |
| `rating` | integer | Yes | 1–5 star rating. |
| `comment` | string | No | Free-text comment. Max length not enforced but keep reasonable. |
| `marketing_opt_in` | boolean | No | Defaults to `true`. Customer consent for future broadcast messages. |

**Response `200 OK` (low rating, no Google Maps link):**
```json
{
  "message": "Thank you for your feedback!",
  "coupon_code": "a3f9b1c2"
}
```

**Response `200 OK` (4–5★ rating with Google Maps configured):**
```json
{
  "message": "Thank you! You're being redirected to leave a public review.",
  "redirect_url": "https://maps.google.com/...",
  "coupon_code": "a3f9b1c2"
}
```

**Errors:**
- `401 Unauthorized` — Invalid or expired token.
- `429 Too Many Requests` — Rate limit exceeded.

## Coupon (`/api/coupon`)

### `POST /api/coupon/redeem`

Redeems a coupon. Validates the staff passcode against the café's bcrypt-hashed password. Idempotent check — already-redeemed coupons return `400`.

**Request Body:**
```json
{
  "coupon_code": "A3F9B1C2",
  "passcode": "staff-secret"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `coupon_code` | string | Yes | The coupon code (case-insensitive). |
| `passcode` | string | Yes | The café's staff passcode. |

**Response `200 OK`:**
```json
{
  "message": "Coupon redeemed successfully.",
  "discount_text": "10% off on your next visit"
}
```

**Errors:**
- `403 Forbidden` — Incorrect passcode.
- `404 Not Found` — Coupon code does not exist.
- `400 Bad Request` — Coupon is already redeemed or expired.

## Admin (`/api/admin`)

All admin endpoints require a valid Supabase JWT (`Authorization: Bearer <token>`). The JWT resolves to the calling café's tenant row.

### `GET /api/admin/me`

Returns the current café owner's profile and subscription state. Used by the frontend's `ProtectedRoute` to check onboarding status.

**Response `200 OK`:**
```json
{
  "id": 42,
  "name": "Blue Tokai Coffee",
  "slug": "cafe-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "subscription_status": "active",
  "subscription_plan": "monthly",
  "plan_expiry": "2026-07-14T10:00:00",
  "marketing_credits": 150,
  "onboarding_completed": true
}
```

**Errors:**
- `401 Unauthorized` — Missing or invalid JWT.
- `404 Not Found` — No café found for this Supabase user (trigger may have failed at signup).

---

### `PATCH /api/admin/me/onboarding`

Completes the one-time onboarding step — sets the café name and marks `onboarding_completed = true`. Can only be called once; subsequent calls return `400`.

**Request Body:**
```json
{ "name": "Blue Tokai Coffee" }
```

**Response `200 OK`:**
```json
{ "message": "Onboarding complete." }
```

**Errors:**
- `400 Bad Request` — Onboarding already completed.
- `401 Unauthorized` — Missing or invalid JWT.

---

### `GET /api/admin/dashboard`

Returns feedback analytics and recent submissions. **Requires an active or trial subscription.**

**Response `200 OK`:**
```json
{
  "total_feedback": 312,
  "average_rating": 4.2,
  "feedbacks": [
    {
      "id": 1,
      "customer_phone": "9876543210",
      "rating": 5,
      "comment": "Best latte in town.",
      "marketing_opt_in": true,
      "created_at": "2026-06-14T10:23:00"
    }
  ]
}
```

**Errors:**
- `401 Unauthorized` — Missing or invalid JWT.
- `403 Forbidden` — Subscription is expired or cancelled.

**Request Body:**

| Field | Type | Required | Values |
|---|---|---|---|
| `plan` | string | Yes | `monthly` (₹999) or `annual` (₹9,999) |

**Response `200 OK`:**
```json

  "amount": 99900,
  "currency": "INR"

```

---

### `POST /api/billing/webhook`

Razorpay payment webhook. Validates the `x-razorpay-signature` HMAC header. Updates the café's subscription to `active` on `payment.captured` events.

> This endpoint is called by Razorpay — not by the frontend.

**Headers:**
- `x-razorpay-signature`: HMAC-SHA256 signature of the raw request body using `RAZORPAY_WEBHOOK_SECRET`.

**Response `200 OK`:**
```json
{ "status": "ok" }
```

**Errors:**
- `400 Bad Request` — Invalid signature.

## WhatsApp (`/api/whatsapp`)

### `GET /api/whatsapp/config`

Returns the WhatsApp Business number used for the QR code deep link. Public — no authentication required.

**Response `200 OK`:**
```json
{ "phone_number": "919876543210" }
```

---

### `GET /api/whatsapp/webhook`

Meta webhook URL verification handshake. Called by Meta when the webhook URL is first configured.

**Query Parameters:**

| Parameter | Description |
|---|---|
| `hub.mode` | Must be `subscribe`. |
| `hub.verify_token` | Must match `META_VERIFY_TOKEN` env var. |
| `hub.challenge` | Random string Meta expects echoed back. |

**Response `200 OK`:** Returns the `hub.challenge` string as plain text.

---

### `POST /api/whatsapp/webhook`

Receives inbound WhatsApp messages from Meta. Protected by HMAC-SHA256 signature verification (`x-hub-signature-256` header).

**Rate limit:** 100 requests / minute.

**Behavior:**
1. Verifies `x-hub-signature-256` header against request body using `META_APP_SECRET`.
2. Deduplicates the message using the `ProcessedWebhook` table.
3. If the message body contains `"ratemyvisit"` or `"rate"`, extracts the `cafe_id` from any digits in the message (defaults to `1`).
4. Generates a customer JWT and sends the feedback link back via WhatsApp text reply.

**Response `200 OK`:**
```json
{ "status": "ok" }
```

**Errors:**
- `400 Bad Request` — Invalid HMAC signature.

## Marketing (`/api/marketing`)

All marketing endpoints require a valid Supabase JWT and an **active or trial subscription**.

### `GET /api/marketing/audience`

Returns the number of opted-in customers and the remaining marketing credit balance.

**Response `200 OK`:**
```json
{
  "audience_size": 87,
  "credits": 150
}
```

---

### `POST /api/marketing/blast`

Sends a WhatsApp template message to all opted-in customers. Deducts one credit per message sent.

**Request Body:**
```json
{
  "template_name": "monthly_offer_v2",
  "components": []
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `template_name` | string | Yes | Must exactly match a **pre-approved** template in Meta Business Manager. |
| `components` | array | No | Template variable substitutions. Format follows Meta's API spec. |

**Response `200 OK`:**
```json
{
  "sent": 85,
  "failed": 2,
  "credits_remaining": 65
}
```

**Errors:**
- `402 Payment Required` — Insufficient credits.
- `403 Forbidden` — Subscription inactive.

## Contact (`/api/contact`)

### `POST /api/contact/`

Submits a contact form message. Public endpoint — no authentication required.

**Rate limit:** 3 requests / hour (per IP).

**Request Body:**
```json
{
  "name": "Ravi Kumar",
  "email": "ravi@example.com",
  "company": "Cafe Chain Co.",
  "phone": "9876543210",
  "message": "Interested in your enterprise plan."
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | Yes | Sender's name. |
| `email` | string | Yes | Valid email address. |
| `message` | string | Yes | The inquiry message. |
| `company` | string | No | Optional company name. |
| `phone` | string | No | Optional phone number. |

**Response `200 OK`:**
```json
{ "message": "Your message has been received. We'll be in touch soon." }
```

**Errors:**
- `422 Unprocessable Entity` — Invalid email format or missing required fields.
- `429 Too Many Requests` — Rate limit exceeded.
